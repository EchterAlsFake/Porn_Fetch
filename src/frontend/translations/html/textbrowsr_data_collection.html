Data Collection & Privacy Information

This application now uses my own server for update checking and error reporting, instead of relying on GitHub. This gives me greater control over the process and the data transmitted. However, my server is IPv6-only. This means that only about 50% of internet users will be able to connect.

About the Server

The server is a small, older Acer Swift 3 laptop running 24/7 in my room. The full source code for the server is publicly available here:
https://github.com/EchterAlsFake/Server

Data Collected via Error Reports

  • The version of Porn Fetch you are using
  • Your operating system (e.g., Windows, Linux, or macOS)
  • The current date and time
  • The error details (full Python traceback)

Important: The Python traceback may, in some cases, include incidental personal information — for example, your system username if it appears in a file path. No other personal data is intentionally collected.

Data Storage & Security

Error reports are stored in plain text on my server. The server’s storage device is encrypted with LUKS and secured with a strong password (40+ characters). Your IP address is never logged, stored, or displayed.

Optional Participation

Whether you enable error reporting or not will have no impact on the application's functionality. It simply helps me identify and fix issues faster. You can also manually check for updates on GitHub, although most users do not do this.

IPv6 Connectivity Check

To see if you have IPv6 connectivity, visit:
https://echteralsfake.duckdns.org/ping
If you see a white page with Success, you have IPv6. If not, you do not.

You can also verify your IP addresses by visiting: https://ipleak.net
